<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User clipboard/paste -> smsText | Unvalidated text from user paste enters generateDraft rawText |
| SMS ContentProvider -> inbox reader | Raw SMS body text from device ContentProvider (feature-flagged path only) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-35-02-01 | Tampering | smsText field | accept | Text is authored/selected by the user themselves — no untrusted third-party input |
| T-35-02-02 | Information Disclosure | SMS_INBOX_ENABLED = false | mitigate | Feature flag is false in active build — READ_SMS is never requested; no inbox data accessed |
| T-35-02-03 | Spoofing | SMS sender display | accept | SMS sender field is displayed as-is for user reference; no authentication decisions are made from it |
| T-35-02-04 | Denial of Service | AI Fill concurrent taps | mitigate | Button disabled while uiState.isGenerating is true — concurrent generateDraft calls blocked |
| T-35-02-05 | Information Disclosure | saveAsNote toggle | accept | SMS text is user-owned content written to their own note field — no third-party data disclosure |
</threat_model>
