**States:**

| State | Condition | Visual |
|-------|-----------|--------|
| Default | Screen opened, no input | Empty text field, "AI Fill" disabled, inbox section shown if READ_SMS granted |
| Text entered | User pastes/types text | Text field populated, "AI Fill" enabled (if AI available) |
| Inbox loading | READ_SMS granted, fetching | `CircularProgressIndicator` replaces list |
| Inbox loaded | Messages available | SMS rows in `LazyColumn`, date-descending, max 50 |
| Inbox empty | No matches for filter | Empty state illustration + copy |
| Row selected | User taps SMS row | Full preview pane animates in (bottom half, `surface` background with `primaryContainer` highlight) |
| AI generating | User taps "AI Fill" | "AI Fill" shows `CircularProgressIndicator` replacement, button disabled |
| AI success | Draft generated | Navigate to `AddEditTransactionDialog` via `onNavigateToConfirm` callback |
| AI failure | Draft generation error | Snackbar: "Could not generate draft. Please enter details manually." (SMS-09) |
| AI unavailable | `isAiAssistAvailable = false` | "AI Fill" button absent entirely (SMS-08), manual text entry only |
| Save as note | Toggle ON | SMS source text will persist to transaction `note` field |
| Save as note | Toggle OFF (default) | SMS text discarded after draft generated |

```
┌─────────────────────────────────────────────┐
│ ← AI Draft from Receipt               28sp  │  TopAppBar
├─────────────────────────────────────────────┤
│  ┌─ Image area (300dp max height) ───────┐  │
│  │  [Thumbnail of captured/selected img] │  │  OCR-04

│                                             │
│  [📷 Take Photo]  [🖼 Gallery]              │  OCR-01
│                                             │
│  ┌─ OCR Text (scrollable, read-only) ────┐  │  OCR-04
│  │  Purchased item 1: ₹1,500.00          │  │
│  │  Purchased item 2: ₹2,300.00          │  │
│  │  Total: ₹3,800.00                     │  │
│  │  Merchant: XYZ Store                  │  │
│  └────────────────────────────────────────┘  │
│                                             │
│  [AI Fill]                                   │  OCR-08
│                                             │
│  Empty state (no image):                     │
│  ┌─ Icon ───────────────────────────────┐   │
│  │ "No receipt captured" 22sp          │   │
│  │ Take a photo of your receipt or     │   │
│  │ select one from your gallery.       │   │
│  └──────────────────────────────────────┘   │
└─────────────────────────────────────────────┘
```

**States:**

| State | Condition | Visual |
|-------|-----------|--------|
| Default | Screen opened, no image | Empty state illustration + copy, "Take Photo" and "Gallery" buttons, "AI Fill" disabled |
| Camera capture | User takes photo | Image thumbnail shown, local URI stored via `FileHelper.saveReceipt()`, OCR auto-triggers on `Dispatchers.IO` (OCR-02, OCR-05) |
| Gallery select | User picks image | Same as camera capture — thumbnail shown, OCR auto-triggers |
| OCR processing | ML Kit analyzing | `CircularProgressIndicator` overlay on image area, text pane shows "Processing..." |
| OCR success | Text recognized | OCR normalized text populates scrollable read-only pane (OCR-03, OCR-04), "AI Fill" enabled (if AI available) |
| OCR failure (1st) | `MlKitException.UNAVAILABLE` | Retry button shown with message: "Could not read text from this image. Tap Retry to try again." (OCR-06) |
| OCR failure (2nd) | Second consecutive failure | Message: "OCR failed again. You can enter the details manually from the visible text." Manual entry path offered (OCR-06) |
| OCR no AI | AI unavailable, OCR succeeded | Text pane remains visible for manual reference, "AI Fill" absent (OCR-09) |
| Camera denied | Permission denied | Camera button hidden, gallery offered as fallback (OCR-07) |
| Gallery unavailable | No gallery app | Screen returns to manual entry instruction |
| AI generating | User taps "AI Fill" | "AI Fill" shows progress indicator |
| AI success | Draft generated | Navigate to `AddEditTransactionDialog` via `onNavigateToConfirm` |
| AI failure | Draft generation error | Snackbar: "Could not generate draft. Please enter details manually." |

**OCR Normalization pipeline (OCR-03):** Applied before text is shown in the pane:
- Rupee symbol variants → `₹`
- Digit–letter confusion (0↔O) resolved in numeric contexts
- Thousands separators normalized
- Boilerplate merchant/header text stripped

### VoiceMemoScreen

```
┌─────────────────────────────────────────────┐
│ ← AI Draft from Voice                 28sp  │  TopAppBar
├─────────────────────────────────────────────┤
│                                             │
│          ┌──────────────┐                    │
│          │   🎤  (88dp) │  ← tap to record   │  VOICE-01
│          │  waveform    │                    │
│          │  ▁▃▄▆▇▆▄▃▁   │  ← animated bars   │
│          └──────────────┘                    │
│                                             │
│          00:42 / 0:60                        │  VOICE-01
│          [Countdown, visible always]         │
│                                             │
│  ┌─ Transcription ───────────────────────┐  │  VOICE-02/03
│  │  [Editable multi-line text field]     │  │
│  │  "Bought groceries for 1500 rupees    │  │
│  │   at Big Bazaar using UPI"            │  │
│  └────────────────────────────────────────┘  │
│                                             │
│  [Re-record]    [AI Fill]                    │  VOICE-04/07
│                                             │
│  [Save transcription as note]  [ON]          │  VOICE-10
│                                             │
│  Empty state (no recording):                 │
│  ┌─ Icon ───────────────────────────────┐   │
│  │ "No voice memo recorded" 22sp       │   │
│  │ Tap the microphone and speak clearly│   │
│  │ to record a transaction description.│   │
│  └──────────────────────────────────────┘   │
└─────────────────────────────────────────────┘
```

**States:**

| State | Condition | Visual |
|-------|-----------|--------|
| Default | Screen opened, no recording | Empty state illustration + copy, mic button idle (no animation), transcription field empty, "Re-record" hidden, "AI Fill" disabled, toggle shown |
| Recording active | User tapped mic | Mic shows waveform animation (`RecognitionListener.onRmsChanged` drives bar heights), countdown counts up from 0:00 to 0:60, transcription field shows "Listening..." placeholder, mic button acts as stop |
| Recording stopped | User tapped mic / 60s reached | Mic returns to idle state, waveform stops, `SpeechRecognizer` processes audio, transcription populates |
| Recording limit | 60 seconds reached | Recording auto-stops, countdown shows "0:60", brief haptic/tactile feedback if available |
| Transcription ready | Speech-to-text complete | Editable transcription field populated (VOICE-03), "Re-record" button visible, "AI Fill" enabled (if transcription non-empty AND AI available) (VOICE-07) |
| Transcription empty | Speech recognized but empty | "AI Fill" disabled, field shows hint: "No speech detected. Tap Re-record or type manually." |
| Re-recording | User taps "Re-record" | Current transcription cleared, new recording session starts (VOICE-04) |
| AI generating | User taps "AI Fill" | "AI Fill" shows progress indicator |
| AI success | Draft generated | Navigate to `AddEditTransactionDialog` via `onNavigateToConfirm` |
| AI failure | Draft generation error | Snackbar: "Could not generate draft. Please enter details manually." |
| AI unavailable | `isAiAssistAvailable = false` | "AI Fill" absent, transcription can be pre-filled as note field in dialog (VOICE-08) |
| Speech unavailable | `isRecognitionAvailable() = false` | Entire screen hidden — voice entry point not shown (VOICE-05) |
| Audio permission denied | `RECORD_AUDIO` denied | Voice entry point hidden, returned to main transaction entry (VOICE-06) |
| Save as note | Toggle ON (default) | Transcription persists to transaction `note` field (VOICE-10) |
| Save as note | Toggle OFF | Transcription discarded after draft generation |

**Waveform Animation Specification (Option B — from user selection):**

| Property | Value |
|----------|-------|
| Component | `Canvas` composable drawing 5-7 vertical bars |
| Bar count | 7 bars, evenly spaced within mic icon container |
| Bar width | 4.dp each, 2.dp gap between |
| Bar height range | 8.dp (idle/min) to 48.dp (max amplitude) |
| Animation driver | `RecognitionListener.onRmsChanged` — RMS value normalized to 0…1 range mapped to bar height |
| Update frequency | Every ~100ms (onRmsChanged callback rate ~10Hz) |
| Smoothing | `animateFloatAsState` with 150ms spring animation on each bar's target height |
| Idle state | All bars at minimum height (8.dp), no animation |
| Active state | Bars oscillate independently using staggered RMS samples from rolling buffer (last 7 RMS values) |
| Color | `MaterialTheme.colorScheme.primary` for all bars |
| Background | `CircleShape` behind mic icon, `surfaceVariant` fill, 88×88.dp |

## Interaction Contract

### Permission Handling (unified pattern across all 3 screens)

| Permission | Flow | Denied Behavior |
|------------|------|-----------------|
| `READ_SMS` | SMS inbox picker (SMS-02) | Feature-flagged; if denied, clipboard/paste-only mode (SMS-06) |
| `CAMERA` | Receipt capture (OCR-01) | Camera button hidden, gallery offered as fallback (OCR-07) |
| `RECORD_AUDIO` | Voice recording (VOICE-01) | Voice entry point hidden entirely (VOICE-06) |

All permission requests use `rememberLauncherForActivityResult(ActivityResultContracts.RequestPermission())`. Permissions requested only on user action (not on screen entry).

### AI Availability Boundary

| State | SmsPickerScreen | ReceiptScanScreen | VoiceMemoScreen |
|-------|----------------|-------------------|-----------------|
| AI available + text ready | "AI Fill" enabled | "AI Fill" enabled | "AI Fill" enabled (transcription non-empty) |
| AI available + no text | "AI Fill" disabled | "AI Fill" disabled | "AI Fill" disabled (transcription empty) |
| AI unavailable | "AI Fill" absent | "AI Fill" absent, OCR text visible for manual ref | "AI Fill" absent, transcription → note field (VOICE-08) |

The `isAiAssistAvailable` boolean is collected from `AiAvailabilityRepository.aiAvailable` Flow (AIFND-02) via `AiDraftViewModel`.

### Navigation Contract

All 3 screens are full-screen composables rendered via navigation routes. Each screen accepts:
- `viewModel: AiDraftViewModel = hiltViewModel()` (shared ViewModel, STD-03)
- `onNavigateBack: () -> Unit` (pops back stack)
- `onNavigateToConfirm: (TransactionDraft) -> Unit` (navigates to confirm/dialog with draft)

No existing file modifications (STD-04). Routes will be added in Phase 36 (DRAFT-08).
