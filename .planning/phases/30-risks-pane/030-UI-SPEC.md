## Color

Material 3 color scheme following 60/30/10 split, inherited from Phase 29 conventions:

| Role | Material 3 Token | Usage |
|------|-------------------|-------|
| Dominant (60%) | surface | Screen background |
| Secondary (30%) | surfaceVariant | Alert card backgrounds (base, before severity tint) |
| Accent (10%) | primary | Pane header title, severity badge text |
| WARNING tint | errorContainer | Severity-tinted card background for WARNING alerts |
| INFO tint | tertiaryContainer | Severity-tinted card background for INFO alerts |
| Destructive | error | Error states only |

Accent reserved for: Pane header title text only (never on alert cards, icons, or explanations)

**Severity tinting rules (from SUMMARY.md Feature Table Stakes):**
- WARNING alerts: card background = `MaterialTheme.colorScheme.errorContainer`
- INFO alerts: card background = `MaterialTheme.colorScheme.tertiaryContainer`
- Card border/elevation: use default Material 3 Card elevation (1dp) for visual separation
- Icon color: inherits the same tint as the card background (onTint token for text/icons on colored surfaces)

## Visuals

### Focal Point
Alert card list — vertically stacked cards ordered by severity (WARNING before INFO), each with icon + title + explanation.

### Visual Hierarchy (Priority Order)
1. **Primary:** RisksPaneHeader ("Risks" heading) + RisksAlertCard items (severity-tinted cards)
2. **Secondary:** HistoryDisclaimer ("Insights improve after your first full month") when `!hasEnoughHistory`
3. **Tertiary:** RisksPaneEmptyState ("No financial activity recorded yet") when no data available
4. **Quaternary:** RisksPaneErrorState (error message + retry) on data load failure

### Layout Rules
- Column layout inside a scrollable container (Column with `verticalScroll`, NOT LazyColumn — avoids scroll interop conflicts inside HorizontalPager per PITFALLS.md)
- Cards stacked vertically with 16dp gap (lg token) between each
- Each card: horizontal row layout with icon (left, 24dp) + text column (title on top, explanation below)
- Card internal padding: 16dp all sides (md token)
- Icon-to-text gap: 12dp (custom — between xs and sm, aligns to 4dp grid)
- HistoryDisclaimer: centered below card list, body typography in `onSurfaceVariant` at 60% opacity
- Empty/error states: fill entire screen, centered vertically and horizontally

### Alert Card Structure (per RSK-07)
```
┌─────────────────────────────────────┐
│ [icon] Title (semibold, 22sp)       │
│        Explanation (regular, 16sp)  │
│        (includes triggering value)   │
└─────────────────────────────────────┘
```

## Interaction Contract

| Interaction | Behavior |
|-------------|----------|
| Scroll | Column scrolls vertically within HorizontalPager page (no nested LazyColumn) |
| Alert ordering | WARNING-severity cards always rendered before INFO-severity cards (enforced by InsightsCalculator.sortedByDescending) |
| Max alerts displayed | 3 (enforced by InsightsCalculator.take(3)) |
| Empty state trigger | `!uiState.hasTransactions` — shows RisksPaneEmptyState instead of alert list |
| History disclaimer trigger | `!hasEnoughHistory` AND `uiState.hasTransactions` — shows disclaimer below cards |
| Error state trigger | Data load failure (separate from empty state) |
