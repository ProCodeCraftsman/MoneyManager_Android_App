## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| AI response → DraftParser | Raw string from LLM crosses into typed domain; adversarial or malformed JSON is possible |
| User master data → PromptBuilder | Category/account/peer names from DB are injected into prompt; injection attack possible if not sanitized |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-33-03 | Tampering | DraftParser — JSON extraction | mitigate | Extract only between first { and last }; catch all deserialization exceptions; return Result.failure instead of crashing |
| T-33-04 | Information Disclosure | PromptBuilder — user data injection | mitigate | Sanitize all name fields: strip double-quotes and newlines before injection to prevent prompt leakage or injection |
| T-33-05 | Denial of Service | DraftParser — very large AI response | accept | Gemini Nano responses are bounded by model token limit (~1024 tokens); no explicit size guard needed at this phase |
| T-33-06 | Spoofing | PromptBuilder — typeId values | accept | TransactionType.allIds() is used as the hint source; DraftParser deserializes typeId as a nullable String; invalid typeId values are handled downstream by the dialog validation layer |
</threat_model>
