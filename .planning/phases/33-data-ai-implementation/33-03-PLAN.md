| AICore availability result → DeviceCapabilityManager | FeatureStatus from ML Kit is trusted; mapping to string constants is deterministic |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-33-08 | Elevation of Privilege | DeviceCapabilityManager — incorrect "READY" status | accept | Status is only used to gate the AI draft feature; a false-positive "READY" causes a graceful API call failure (NanoAiClient returns Result.failure), not a security escalation |
| T-33-09 | Tampering | DeviceCapabilityManager — Boolean written to String key | mitigate | setAiAvailabilityStatus accepts String; compiler enforces type safety; string literals "READY"/"NEVER"/"PENDING" are the only values written |
| T-33-10 | Information Disclosure | NanoAiClient — exception message leakage | accept | AiUnavailableException wraps exception message; this is only logged/shown internally to the developer; no user-facing display of raw exception messages |
</threat_model>
