‚ĚĮ /gsd-new-milestone Integration Blueprint: User‚ÄĎDriven AI‚ÄĎAssisted Transaction 
 Drafting with Gemini Nano (AICore)                                             
Revised Objective: Transform a standard financial tracking app into an          
AI‚ÄĎassisted drafting tool where Gemini Nano (via AICore) helps users quickly    
create transaction drafts from SMS, receipt images, or voice memos. The AI      
suggests a draft; the user always reviews, edits, and confirms. The app remains 
 fully functional without AI ‚Äď all transaction fields can be entered manually.  
                                                                                
Core Principles (unchanged): Clean Architecture, MVVM, Kotlin/Compose, Hilt,    
Room, DataStore, 100% offline, Min SDK 26.                                      
                                                                                
1. Model Selection & Availability Strategy                                      
Model: Gemini Nano accessed exclusively through Android AICore / ML Kit GenAI   
Prompt API.                                                                     
                                                                                
Zero APK bloat ‚Äď model is system‚ÄĎmanaged.                                       
                                                                                
Automatic NPU acceleration on Snapdragon 8 Gen 3 (S24 Ultra).                   
                                                                                
Fully offline, privacy‚ÄĎpreserving.                                              
                                                                                
Compatibility & Fallback:                                                       
                                                                                
AICore is required for AI‚ÄĎassisted drafting. If AICore is not present on the    
device, AI features are disabled and hidden.                                    
                                                                                
The app remains fully operational: users can still manually enter transactions, 
 scan receipts for raw OCR text (but no AI‚ÄĎbased extraction), and use offline   
speech‚ÄĎto‚ÄĎtext for transcription (without AI parsing).                          
                                                                                
This approach satisfies ‚Äúeven without using AI user should be able to use the   
app as it is now.‚ÄĚ                                                              
                                                                                
Capability Detection & Caching:                                                 
DeviceCapabilityManager checks for the AICore package and caches a Boolean      
(isAiAssistAvailable) in DataStore Preferences. All UI decisions (showing ‚ÄúAI   
Fill‚ÄĚ buttons) are driven by a Flow<Boolean> from a repository.                 
                                                                                
2. Architectural Clean Integration (AI as Assistive Service)                    
The AI client remains a data‚ÄĎsource abstraction (GenAiClient) injected via      
Hilt, but it is now used exclusively by domain‚ÄĎlayer use cases that transform   
raw input into a TransactionDraft.                                              
                                                                                
2.1 Domain Layer (Pure Kotlin)

data class TransactionDraft(                                                    
    val transactionType: String?,  // DEBIT / CREDIT                            
    val amount: Double?,                                                        
    val currency: String?,                                                      
    val merchant: String?,                                                      
    val date: LocalDate?,                                                       
    val category: String?,                                                      
    val rawSource: String?        // original SMS/OCR text for user reference

// Use case: generate draft from any text (SMS, OCR output)                     
class GenerateDraftFromTextUseCase @Inject constructor(                         
    private val aiClient: GenAiClient?   // nullable ‚Äď injected only if AI      
available                                                                       
) {                                                                             
    suspend operator fun invoke(rawText: String): Result<TransactionDraft> {    
        if (aiClient == null) return                                            
Result.failure(UnsupportedOperationException())                                 
        return aiClient.generateStructuredDraft(rawText)                        
    }                                                                           
}                                                                               
When AI is disabled, Hilt provides null for GenAiClient, and the use‚ÄĎcase       
simply returns an error. The ViewModel catches this and disables the AI button. 
                                                                                
2.2 Data Layer ‚Äď AI Client Implementation                                       
kotlin                                                                          
interface GenAiClient {                                                         
    suspend fun generateStructuredDraft(inputText: String):                     
Result<TransactionDraft>                                                        
}                                                                               
                                                                                
class NanoAiClient @Inject constructor(                                         
    @ApplicationContext private val context: Context                            
) : GenAiClient {                                                               
    private val model = GenerativeModel(                                        
        modelName = "gemini-nano",                                              
        applicationContext = context                                            
    )                                                                           
    override suspend fun generateStructuredDraft(inputText: String):            
Result<TransactionDraft> {                                                      
        val prompt = buildPrompt(inputText)                                     
        return try {                                                            
            val rawOutput = model.generateContent(prompt).text ?: ""            
            parseDraft(rawOutput)                                               
        } catch (e: Exception) {                                                
            Result.failure(e)                                                   
        }                                                                       
    }                                                                           
    // buildPrompt and parseDraft described in Section 5                        
}

3. User‚ÄĎDriven AI‚ÄĎAssisted Transaction Creation Flows                           
All flows share the same pattern:                                               
                                                                                
User initiates action (select SMS, capture image, record audio).                
                                                                                
Raw data (text/OCR/transcription) is acquired on‚ÄĎdevice.                        
                                                                                
User taps ‚ÄúAI Fill‚ÄĚ ‚Üí raw data sent to Gemini Nano ‚Üí draft returned.            
                                                                                
Draft populates the transaction form; user reviews, edits, and saves.           
                                                                                
No automatic background processing. The app uses simple ViewModel‚ÄĎcoroutine     
interactions.                                                                   
                                                                                
3.1 SMS Flow                                                                    
Acquire text: The user opens a list of recent financial SMS (queried from       
Telephony.Sms inbox, optionally filtered by sender numbers). The user picks a   
message; its body is extracted.                                                 
                                                                                
Display: The raw message is shown in a card.                                    
                                                                                
AI Assist: Tapping ‚ÄúAI Fill‚ÄĚ calls generateDraftFromTextUseCase(smsBody). On    
success, the draft fields are bound to the Compose form.                        
                                                                                
Manual fallback: The user can directly type values.                             
                                                                                
3.2 Receipt / Invoice Scanning (OCR)                                            
Capture: Use ActivityResultContracts.TakePicture or pick from gallery.          
                                                                                
On‚ÄĎdevice OCR: Unbundled ML Kit                                                 
(com.google.android.gms:play-services-mlkit-text-recognition) runs in a         
CoroutineWorker (to avoid blocking UI) and returns raw text.                    
No bundled models ‚Äď zero APK size increase.                                     
                                                                                
AI Assist: The raw OCR text is passed to the same generateDraftFromTextUseCase. 
 The prompt instructs the model to handle noisy OCR output (e.g., ‚ÄúT0TAL‚ÄĚ,      
mixed currencies).                                                              
                                                                                
Without AI: The raw OCR text is displayed for the user to read and manually     
transcribe.                                                                     
                                                                                
3.3 Voice Memo Flow                                                             
Recording: Use MediaRecorder to capture audio into a local file.                
                                                                                
Speech‚ÄĎto‚ÄĎText: Use Android‚Äôs built‚ÄĎin SpeechRecognizer with                    
EXTRA_PREFER_OFFLINE = true. This guarantees on‚ÄĎdevice transcription (privacy), 
 though accuracy may vary. The result is a plain‚ÄĎtext string.                   
                                                                                
AI Assist: The transcribed text is fed to generateDraftFromTextUseCase. The     
prompt instructs the model to extract financial fields from natural language    
(e.g., ‚ÄúI spent 45 euros at Lidl yesterday‚ÄĚ).                                   
                                                                                
Without AI: The transcription is shown as a note; user fills fields manually.   
                                                                                
4. System Robustness, Fallbacks & Graceful Degradation                          
AI Unavailability Handling:                                                     
                                                                                
If AICore is not detected at install/launch, isAiAssistAvailable = false is     
persisted.                                                                      
                                                                                
All ‚ÄúAI Fill‚ÄĚ buttons are visibility = if (aiAvailable) Visible else Gone.      
                                                                                
Manual entry remains the primary interface; the app is never broken.            
                                                                                
Runtime AI Errors:                                                              
If an AI call fails (network‚ÄĎrelated? no, it‚Äôs offline, but AICore may throw    
exceptions for unsupported requests), the ViewModel catches the error and shows 
 a Snackbar: ‚ÄúCould not generate draft. Please enter details manually.‚ÄĚ The     
form is cleared to default state.                                               
                                                                                
Memory & Performance:                                                           
Gemini Nano runs in the Private Compute Core ‚Äď it does not consume the app‚Äôs    
heap. Thus, even on 4‚ÄĮGB devices with AICore, memory pressure is minimal. No    
need for custom memory management.                                              
                                                                                
DataStore Caching:                                                              
DeviceCapabilityManager uses DataStore to store the isAiAssistAvailable flag.   
This is read once per app session; no repeated probing.                         
                                                                                
5. Deterministic Prompting & Data Recovery                                      
Gemini Nano must return a predictable JSON object. The prompt is engineered as  
a single‚ÄĎshot, strict instruction.                                              
                                                                                
System Prompt (embedded in the generative model call):                          
                                                                                
text                                                                            
You are an AI transaction drafter. Extract the following fields from the given  
text and output ONLY a JSON object.                                             
Fields: transaction_type ("DEBIT","CREDIT","UNKNOWN"), amount (number),         
currency (ISO 4217), merchant (string or null), date (YYYY-MM-DD or null),      
category (one of                                                                
"Food","Transport","Shopping","Utilities","Entertainment","Transfer","Other" or 
 null).                                                                         
If a field cannot be inferred, use null.                                        
Do NOT include markdown or explanations.                                        
                                                                                
Input text: {RAW_TEXT}                                                          
JSON:                                                                           
Defensive Parsing:                                                              
The parseDraft method applies the same multi‚ÄĎstage recovery described earlier   
(JSON boundary extraction, kotlinx.serialization with ignoreUnknownKeys = true, 
 regex fallback). If parsing fails, an error is returned to the user.           
                                                                                
Note: Because the AI only suggests a draft, imperfect extractions are           
acceptable. The user will correct any errors before saving.
